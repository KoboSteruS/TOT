# 📁 Структура проекта ООО «ТОТ»

## Общее описание

Сайт **ООО «ТОТ»** — это современный одностраничный (Single Page) лендинг для бухгалтерской компании с интегрированным чат-ботом и полноценной админ-панелью для управления контентом.

## Дерево файлов проекта

```
TOT/
├── app.py                          # Основное Flask-приложение (бэкенд)
├── requirements.txt                # Python-зависимости
├── .env                           # Переменные окружения (секреты)
├── .env.example                   # Пример файла .env
├── .gitignore                     # Исключения для Git
├── README.md                      # Основная документация проекта
├── CERTIFICATES.md                # Документация по сертификатам
├── start.bat                      # Скрипт запуска (Windows)
├── start.sh                       # Скрипт запуска (Linux/Mac)
│
├── templates/                     # HTML-шаблоны
│   ├── index.html                 # Главная страница сайта (публичная)
│   └── admin.html                 # Админ-панель
│
└── static/                        # Статические файлы
    ├── styles.css                 # Все стили сайта (3668 строк)
    ├── content.json               # Контент сайта (управляется через админку)
    ├── faq.json                   # База знаний чат-бота
    ├── certificates.json          # Сертификаты компании
    │
    └── images/                    # Изображения
        ├── Logo.png               # Логотип компании
        ├── Hero.png               # Фото героя (главный экран)
        ├── about.png              # Фото раздела "О нас"
        ├── cta-laptop.png         # Ноутбук для CTA-блока
        ├── tariffs-bg.png         # Фон тарифов
        └── partners/              # Изображения партнёров
            ├── alfa-banner.png    # Баннер Альфа-Банк
            └── alfa-certificate.png # Сертификат партнёрства
```

## Основные компоненты

### 1. **Backend (Flask)**
- **Файл**: `app.py`
- **Размер**: ~14KB
- **Функции**:
  - Рендеринг публичных страниц (`index.html`, `admin.html`)
  - REST API для управления контентом
  - JWT-авторизация для админ-панели
  - Загрузка и управление изображениями
  - CRUD операции для FAQ, сертификатов, контента

### 2. **Frontend (HTML/CSS/JS)**
- **Публичная страница**: `templates/index.html` (~85KB)
  - Одностраничный сайт с секциями
  - Встроенный чат-бот
  - Модальные окна
  - Адаптивная вёрстка
  
- **Админ-панель**: `templates/admin.html` (~71KB)
  - Тёмная тема
  - Управление всем контентом сайта
  - Загрузка изображений
  - Управление FAQ и сертификатами

- **Стили**: `static/styles.css` (~65KB, 3668 строк)
  - Единый файл стилей
  - Mobile-first подход
  - Множество медиа-запросов
  - Анимации и переходы

### 3. **Data (JSON-файлы)**
- `content.json` (~15KB) — весь текстовый контент сайта
- `faq.json` (~53KB) — база знаний бота (386+ строк)
- `certificates.json` (~335 байт) — данные сертификатов

## Технологический стек

### Backend
- **Python 3.x**
- **Flask** — веб-фреймворк
- **PyJWT** — JWT-токены для авторизации
- **python-dotenv** — управление переменными окружения
- **Werkzeug** — безопасная работа с файлами

### Frontend
- **HTML5** — семантическая разметка
- **CSS3** — современные стили (Grid, Flexbox, Custom Properties)
- **Vanilla JavaScript** — без фреймворков
- **Yandex Maps API** — карта с офисом

### Инфраструктура
- **Git** — контроль версий
- **Environment Variables** — безопасное хранение секретов
- **JSON** — хранение данных

## Особенности архитектуры

### 1. **Разделение данных и представления**
Весь контент вынесен в JSON-файлы, что позволяет:
- Редактировать сайт без изменения кода
- Управлять через админ-панель
- Легко добавлять переводы (i18n в будущем)

### 2. **Модульность**
- Секции сайта независимы друг от друга
- Каждый блок можно редактировать отдельно
- API построено по RESTful принципам

### 3. **Безопасность**
- JWT-авторизация для админки
- Секреты в `.env` файле
- Валидация загружаемых файлов
- CSRF-защита через токены

### 4. **Производительность**
- Единый CSS-файл (минимум запросов)
- Оптимизированные изображения
- Ленивая загрузка данных через API
- Кеширование статических файлов
