# 🔐 Админ-панель ООО «ТОТ»

## Общее описание

Админ-панель — это полнофункциональный интерфейс для управления **всем контентом сайта** без необходимости редактирования кода.

**URL доступа**: `http://ваш-домен/admin/{JWT_TOKEN}`

**Файл**: `templates/admin.html` (~71KB, 2500+ строк)

---

## 🚪 Вход в админ-панель

### Генерация токена доступа

1. **Через переменные окружения** (`.env`):
```env
ADMIN_TOKEN=ваш-секретный-токен
JWT_SECRET=ваш-jwt-секрет
```

2. **Для разработки** (временный эндпоинт):
```
GET /api/generate-token
```
Возвращает JWT-токен для доступа.

⚠️ **ВАЖНО**: В production удалите эндпоинт `/api/generate-token`!

### URL доступа

```
http://localhost:5000/admin/ваш-jwt-токен
```

Пример:
```
http://localhost:5000/admin/eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

При правильном токене откроется админ-панель с тёмным интерфейсом.

## 3️⃣ FAQ (Управление чат-ботом)

### Структура FAQ

Каждый вопрос (интент) содержит:
```json
{
  "intent": "unique_intent_name",
  "keywords": ["ключ1", "ключ2", "..."],
  "response": "Ответ бота",
  "category": "general|services|tariffs",
  "price": "от 10 000 ₽" (опционально)
}
```

### Список FAQ

Таблица со всеми вопросами:
- **Intent** — уникальный ID вопроса
- **Keywords** — ключевые слова для поиска
- **Response** — текст ответа (обрезан для компактности)
- **Category** — категория
- **Действия**: Редактировать | Удалить

### Добавление нового вопроса

1. Нажмите **"Добавить вопрос"**
2. Заполните форму:
   - **Intent**: уникальное имя (например, `new_service`)
   - **Keywords**: ключевые слова через запятую
   - **Response**: текст ответа бота
   - **Category**: выберите категорию
   - **Price** (опционально): цена услуги
3. Нажмите **"Сохранить"**

### Редактирование вопроса

1. Нажмите **"Редактировать"** у нужного вопроса
2. Измените данные в форме
3. Нажмите **"Сохранить"**

### Удаление вопроса

1. Нажмите **"Удалить"**
2. Подтвердите действие (появится alert)

### Категории FAQ

- **general** — общие вопросы (о компании, контакты, гарантии)
- **services** — услуги (бухучёт, налоги, кадры, регистрация)
- **tariffs** — тарифы и цены

### Как работает поиск

Бот ищет совпадения в массиве `keywords`. Если нашёл хотя бы одно слово — возвращает `response`.

**Пример**:
```
Пользователь: "Сколько стоит бухгалтерия?"
Keywords: ["бухгалтерия", "стоимость", "цена"]
→ Бот возвращает ответ по интенту "accounting_service"
```

- Форматы: JPG, PNG, GIF, SVG, WebP
- Максимальный размер: 16MB

### Замена изображения на сайте

1. Нажмите **"Replace"** у нужного изображения (например, Logo.png)

3. **ВАЖНО**: Файл должен иметь **то же имя** (например, `Logo.png`)
4. Файл будет загружен и **перезапишет** старый
5. Обновите страницу — изменения применятся мгновенно

⚠️ **Внимание**: При замене старый файл удаляется!

### Загрузка нового изображения

1. Перетащите файл в зону **"Upload New Image"**
   - Или кликните и выберите файл
2. Файл загрузится с уникальным именем (timestamp)
3. Появится в списке **"Uploaded Images"** ниже

### Список загруженных изображений

Показывает все файлы из `static/images/`:
- Превью изображения
- Имя файла

- Кнопка **"Delete"** (удалить)

- `content.json` (например, для баннера партнёра)

## 🔌 API для админ-панели

Все операции выполняются через REST API с JWT-авторизацией.

### Авторизация

Каждый запрос должен содержать header:
```
Authorization: Bearer {JWT_TOKEN}
```

### Эндпоинты

#### Content API

```http
GET /api/content
```
Получить весь контент сайта.

```http
PUT /api/content
```
Обновить весь контент.

```http
GET /api/content/{section}
```
Получить конкретную секцию (например, `/api/content/hero`).

```http
PUT /api/content/{section}
```
Обновить конкретную секцию.

---

#### FAQ API

```http
GET /api/faq
```
Получить все вопросы.

```http
POST /api/faq
```
Добавить новый вопрос.

Body:
```json
{
  "intent": "new_question",
  "keywords": ["ключ1", "ключ2"],
  "response": "Ответ",
  "category": "general"
}
```

```http
PUT /api/faq/{index}
```
Обновить вопрос по индексу.

```http
DELETE /api/faq/{index}
```
Удалить вопрос по индексу.

---

#### Certificates API

```http
GET /api/certificates
```
Получить все сертификаты.

```http
POST /api/certificates
```
Добавить новый сертификат.

Body:
```json
{
  "title": "Название",
  "image": "/path/to/image.png",
  "description": "Описание",
  "date": "2024"
}
```

```http
PUT /api/certificates/{cert_id}
```
Обновить сертификат.

```http
DELETE /api/certificates/{cert_id}
```
Удалить сертификат.

---

#### Images API

```http
POST /api/upload
```
Загрузить изображение.

Form-data:
- `file`: файл изображения
- `path` (опционально): путь для замены существующего файла

```http
GET /api/images
```
Получить список всех изображений.

Response:
```json
{
  "images": [
    {
      "name": "image.png",
      "url": "/static/images/image.png",
      "size": 12345
    }
  ]
}
```

```http
DELETE /api/images/{filename}
```
Удалить изображение.

## 💡 Tips & Tricks

### 1. Резервное копирование

Перед масштабными изменениями сделайте копию JSON-файлов:
```bash
cp static/content.json static/content.json.backup
cp static/faq.json static/faq.json.backup
cp static/certificates.json static/certificates.json.backup
```

### 2. Тестирование изменений

После изменения контента:
1. Сохраните в админке
2. Откройте публичную страницу в новой вкладке
3. Обновите страницу (Ctrl+F5 для жёсткого обновления)

### 3. Форматирование текста

В полях `description` можно использовать:
- `<br>` — перенос строки
- `<strong>` — жирный текст
- Но не переусердствуйте с HTML!

### 4. Оптимизация изображений

Перед загрузкой:
- Сжимайте изображения (TinyPNG, ImageOptim)
- Используйте WebP для лучшего сжатия
- Рекомендуемые размеры:
  - Logo: 200x60px
  - Hero: 1920x1080px
  - About: 800x600px
  - Сертификаты: 800x1200px

### 5. Безопасность

- **Никогда** не делитесь JWT-токеном
- Меняйте `JWT_SECRET` регулярно
- В production отключите `/api/generate-token`
- Используйте HTTPS для доступа к админке

## 🚨 Troubleshooting

### Не открывается админ-панель

1. Проверьте правильность токена
2. Убедитесь, что Flask запущен
3. Проверьте `.env` файл (JWT_SECRET должен совпадать)

### Изменения не сохраняются

1. Проверьте консоль браузера (F12) на ошибки
2. Убедитесь, что у Python есть права на запись в `static/`
3. Проверьте, что Flask запущен не в read-only режиме

### Изображения не загружаются

1. Проверьте размер файла (<16MB)
2. Проверьте формат (только JPG, PNG, GIF, SVG, WebP)
3. Убедитесь, что папка `static/images/` существует и доступна для записи

### Бот не отвечает на новые вопросы

1. Проверьте, что FAQ сохранился в `static/faq.json`
2. Обновите страницу с ботом (чат загружает FAQ при открытии)
3. Проверьте правильность keywords (должны совпадать со словами в вопросе)
